* Pro-Tips for Total Mouse Independence

** The Grid Editing Speedrun
When editing tables directly via the data grid, you can press ~Alt-Insert~ to append rows, type your data, hit ~Tab~ to jump columns, and instantly commit it to the database with ~Ctrl-S~ without ever reaching for your mouse to click toolbars.

** The Cursor Execution Trick
You do not need to manually highlight a query block to run it. As long as your cursor is sitting anywhere inside the statement, ~Ctrl-Enter~ executes just that specific block (isolated by semicolons or blank lines).

** Customizing & Fixing Linux Window Manager Clashes
Because Linux window managers often intercept keys like ~Alt-F4~ or ~Ctrl-Alt-Space~, you might hit a collision. Press =Ctrl-3=, type "Keys", and press ~Enter~ to jump straight to the key-binding settings panel to remap them on the fly.

* Workspace Presentation Guide

** Record View vs. Data Grid
The Record view is incredibly useful if your table contains dozens of columns. By switching to it, DBeaver flips the orientation vertically, allowing you to view a single row's attributes as a clean list. This makes it convenient to copy data from one record to another without horizontal scrolling fatigue.

To open this view, simply select one or several rows and press ~Tab~ on your keyboard. Press ~Tab~ again to instantly jump back to the standard horizontal Data Grid.

** Plain Text Presentation
Besides the structured table view, you can view your Data Grid output in a raw, plain text format styled with a clean monospace font. This is useful when table cells contain extra-long strings or JSON payloads, allowing you to see the text limits clearly or copy structured output segments precisely.

To cycle into this view, simply press =Ctrl-`=. Press it again to return to the interactive table cells.
